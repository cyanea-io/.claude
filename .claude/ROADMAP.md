## Principles

1. **Keep rules focused** - One concern per file
2. **Make rules actionable** - Specific guidance over general advice
3. **Version everything** - Track changes over time
4. **Test rules in practice** - Validate before standardizing
5. **Iterate based on feedback** - Rules should evolve
